Practice 1

请描述页目录项（Page Directory Entry）和页表项（Page Table Entry）中组成部分对ucore实现页替换算法的潜在用处。

lab2 的复刻，建议 lab3 和 lab2 合并嗷。

可以利用 PDE，PDT 记录某一个页的状态信息，比如 A D 等，利用这些信息可以实现后面的 extended_clock。

如果ucore的缺页服务例程在执行过程中访问内存，出现了页访问异常，请问硬件要做哪些事情？

见 lab2 ，不写了。

Practice 2

如果要在ucore上实现”extended clock页替换算法”请给你的设计方案，现有的swap_manager框架是否足以支持在ucore中实现此算法？如果是，请给你的设计方案。如果不是，请给出你的新的扩展和基此扩展的设计方案。并需要回答如下问题

需要被换出但页但特征是什么？
在ucore中如何判断具有这样但特征页？
何时进行换入和还出操作？

  • 可以
  • 按优先级由高到低，没被访问也没被修改，被访问但没被修改，访问且修改。
  • 缺页异常时换入，物理内存满且有新页换入时换出。

Ex Challenge

实现识别dirty bit的 extended clock页替换算法（需要编程）

基本的 clock 算法是利用 R 位然后从最老到最新查找 R 位为 0 的，并在查找途中把 R 位清 0。

如果要实现识别 dirty bit 就参考上面的优先级换出页面即可。
